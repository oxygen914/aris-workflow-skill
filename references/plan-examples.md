# 计划书示例

本文档提供完整的论文研究计划书示例模板，包括两种典型案例。

> 注：以下示例中的路径使用占位符表示。实际使用时，用户应替换为真实路径。

---

## 示例1：技术类论文计划书模板

**项目名称**: 基于 FaaS 与任务 DAG 封装的智能救灾应用快速响应方法研究

**计划书路径**: `<output-root>/计划书-jsa.md`

### 核心特点

1. **强约束绑定**: 强制性遵循框架文档的所有内容
2. **七章结构**: 标准的学术论文章节组织
3. **三个创新点**: 明确的创新贡献定义
4. **五阶段工作流**: 从规划到改进的完整流程

### 适用场景

- 有完整技术框架文档的研究
- 创新点明确的技术方法研究
- 需要严格遵循规范的技术论文

### 关键配置片段

```yaml
constraint:
  source_file: "<research-root>/论文/技术框架文档.md"
  binding_level: "mandatory"
  validation_mode: "strict"
  
  requirements:
    - "所有章节内容必须严格遵循框架文档的结构和要求"
    - "所有创新点表述必须使用框架文档中的原文定义"
    - "所有技术方案必须按照框架文档中的描述实现"
```

### 工作流映射

```yaml
workflow_sequence:
  - skill: paper-plan
    role: component
    description: "生成详细论文规划"
  - skill: paper-figure
    role: component
    description: "生成论文图表"
  - skill: paper-write
    role: component
    description: "撰写论文各章节"
  - skill: paper-compile
    role: component
    description: "编译 LaTeX 论文"
  - skill: auto-review-loop
    role: component
    description: "自动改进循环"
```

### 阶段定义特点

每个阶段包含：
- 执行指令
- 输入输出定义
- 质量检查函数
- 失败处理方案
- 框架一致性验证

### 质量门禁

```yaml
quality_gates:
  - stage: 1
    checks:
      - metric: "chapter_coverage"
        threshold: 1.0
      - metric: "experiment_coverage"
        threshold: 0.8
    action_on_fail: "block"
```

## 示例2：实证研究类论文计划书模板

**项目名称**: 基于观察与评估量表的多届次获奖作品研究

**计划书路径**: `<output-root>/计划书-cao.md`

### 核心特点

1. **基准研究延伸**: 在既有研究基础上扩展研究范围
2. **量表设计核心**: 以观察量表和评估量表构建为核心方法
3. **数据采集复杂**: 需要多源数据采集和处理
4. **四工作流组合**: Workflow 1 -> 1.5 -> 2 -> 3

### 适用场景

- 既有研究的延伸和扩展
- 需要数据采集和文本分析的研究
- 评估工具构建类研究
- 跨届次比较研究

### 关键配置片段

```yaml
mandatory_constraints:
  research_direction_binding:
    source_file: "<research-root>/研究方向.md"
    requirements:
      - "论文研究主线必须围绕研究对象展开"
      - "论文重点必须放在量表设计与评估量表构建上"
  
  baseline_binding:
    source_file: "<research-root>/基准研究论文.pdf"
    binding_level: "reference_and_extension"
    requirements:
      - "必须复核基准论文的研究对象和范围"
      - "不得直接假定新样本可用，必须重新核验"
```

### 研究边界定义

```text
主线定位:
本研究以特定类别获奖作品为研究对象，在既有研究的基础上将样本核验范围扩展至更多届次。研究重点不是单纯统计样本表现，而是先梳理相关研究、评估标准和参考框架，据此设计观察量表，进一步构建具有评分等级、评分说明和证据要求的评估量表。

关键链条:
1. 延伸研究必须先复核样本，再谈分析。
2. 样本复核依赖官方获奖名单、作品页面、视频链接和作品元数据。
3. 观察量表和评估量表是研究设计的核心。
...
```

### 输出目录结构

```text
<output-root>/
├── 00-项目清单/
├── 01-基准论文复核/
├── 02-获奖名单核验/
├── 03-视频与字幕/
├── 04-台词文本/
├── 05-词频图表/
├── 06-观察量表与作品分析/
├── 07-论文写作/
├── 08-质量验证/
└── 09-调度日志/
```

### 阶段定义特点

- **阶段0**: 项目初始化与环境盘点
- **阶段1**: 研究方向与基准论文复核
- **阶段2**: 样本核验与范围确认
- **阶段3**: 视频下载与字幕获取能力补齐
- **阶段4**: 文本提取、转写与人工校对
- **阶段5**: 词频统计与图表绘制
- **阶段6**: 量表设计、评估量表构建与评分
- **阶段7**: 图表整合与研究发现生成
- **阶段8**: 论文规划与正文写作
- **阶段9**: 自动审校与迭代改进

### 工具能力缺口处理

```yaml
tooling_gap:
  problem: "视频下载和文本提取方案需要验证"
  required_handling:
    - "不得把视频下载和文本提取步骤写成已完成"
    - "必须单列能力补齐阶段，由 Agent 先检测工具，再选择可执行路径"
    - "若无法自动转写，必须生成手动转写/人工校对任务单"
```

## 两种计划书对比

| 维度 | 技术类计划书 | 实证研究类计划书 |
|---|---|---|
| 约束类型 | 框架文档强制绑定 | 研究方向+基准研究双重绑定 |
| 工作流阶段数 | 5阶段 | 10阶段 |
| 数据采集 | 无需外部数据采集 | 需要多源数据采集 |
| 方法核心 | 技术方案实现 | 量表设计与评估 |
| 工具依赖 | LaTeX编译工具 | 视频下载、ASR转写、词频统计 |
| 质量门禁重点 | 框架一致性 | 数据完整性和量表可操作性 |
| 失败处理重点 | 回归框架原文 | 记录能力缺口和人工介入 |

## 选择建议

### 选择技术类模板的情况

- 有完整的技术框架文档
- 创新点和方法论已经明确定义
- 需要严格遵循特定结构
- 主要是写作和实现工作
- 不涉及大规模数据采集

### 选择实证研究类模板的情况

- 在既有研究基础上延伸
- 需要构建评估工具或方法
- 涉及多源数据采集和处理
- 研究对象范围需要核验
- 有工具能力缺口需要处理

## 使用说明

1. **确定研究类型**: 根据研究特点选择合适的模板风格
2. **收集关键信息**: 研究方向、基准研究、约束条件、输出要求
3. **生成计划书**: 使用 aris-workflow-skill 生成完整计划书
4. **验证完整性**: 检查所有必需模块是否存在
5. **调整细节**: 根据具体研究调整阶段定义和配置
6. **执行工作流**: 按照计划书执行研究工作流
